Why need CI & CD
Continios integration-епрерывная интеграция-интеграцию модулей системы мы делаем постоянно(без каких-то модулей-> новая фича-новая интеграция).
мы раньше узнаем о дефектах.
Требования ci:
1. Version control - Код должен хранится в нейком гите.
2. Build automation - Сборка автоматизирована.
3. Test Automation
4. Нужно делать часто комиты
5. Как только у нас изменение начинается ci
Зачем?
1. Чтобы в конце иметь протестированное приложение 
+:
1. Рано обнаруживаем дефекты, потому их дешевле исправить
2. Имеем возможность автоматически установить приложение
3. Прозрачный монитор здоровья приложения
4. Есть возможность визуализировать цепочку сбора
5. Есть возможность распапалелить задачи.

Why Jenkins?
Jenkins-самый популярная и хорошая платформа.
Он бесплатный и поддерживает команды. Кроссплатформенный. Модульная система, много плагинов.

Build triggers
наиболее используемые:
1. Запуск по клику
2. Запус задачи после завершения другой
3. По рассписанию
4. SCM check
